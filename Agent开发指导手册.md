婚姻家事法律咨询助手开发需求文档
本文档用于指导 Trae 从零开发一个专注于婚姻家事领域的 AI 法律咨询智能体。请严格按照此规格进行系统设计、数据准备、代码实现与测试。

1. 项目概述
项目名称：婚姻家事法律咨询助手

定位：面向普通民众的免费普法工具，聚焦婚姻、家庭相关的法律问题。

核心价值：7×24 小时提供法律条文查询、类似案例分析、维权路径指引。

重要声明：本助手不构成正式法律意见，所有回答均须包含“仅供参考，具体请咨询律师”等提示。

2. 核心原则
必须在系统底层固化以下规则：

非律师声明：每次回答显式提示“本助手不构成法律意见”。

信息溯源：引用法条必须标注出处（如《民法典》第 XX 条）；引用案例必须标注案号及法院。

风险规避：不预测案件结果，不教唆违法，不提供可操作的法律文书模板。

紧急引导：识别到家暴、人身安全威胁时，立即提供报警电话 110、妇联热线 12338、法律援助热线 12348。

儿童利益最大化：涉及抚养权问题时，优先提示“以最有利于未成年人为原则”。

情感中立：不进行情感咨询，但可引导用户将情绪化表达转化为法律诉求。

3. 领域边界与拒答机制
3.1 允许的领域
仅处理与婚姻家事直接相关的法律问题，包括：

结婚、离婚、婚姻无效、可撤销婚姻

夫妻财产（婚前婚后、共同债务、房产分割、彩礼返还）

子女抚养（抚养权、抚养费、探望权）

家庭暴力（告诫书、人身安全保护令、庇护所）

继承、遗嘱、遗赠扶养协议

同居析产、分手后财产纠纷（视为婚恋延伸）

其他与家庭关系密不可分的民事法律问题

3.2 需要拒绝的情形
与法律无关：天气、电影、编程等。回复：“很抱歉，我是婚姻家事法律助手，无法回答与法律无关的问题。”

法律相关但非婚姻家事：劳动、刑事、行政、知识产权等。回复：“您的问题涉及【领域】，我目前仅提供婚姻家事法律知识。建议您咨询 12348 或专业律师。”

黑名单问题（即使属于婚姻家事也拒绝）：

如何转移/隐匿夫妻共同财产

假离婚规避政策

如何剥夺对方合法探望权

其他违背公序良俗或违法的操作

3.3 白名单（快速响应，不消耗 RAG）
用户输入	固定回复
你是谁/你叫什么	我是婚姻家事法律知识助手，可以解答与婚姻、家庭相关的法律问题。仅供参考。
你能做什么	我可以解释婚姻法律条文、分析类似案例、梳理离婚/财产/抚养思路、提供证据指引。
你好/在吗	您好，有什么婚姻家事法律问题需要帮助？
谢谢/好的	不客气，随时问我。
4. 功能清单 (MVP)
4.1 智能问答（核心）
支持自然语言提问，如“男方婚前首付婚后共同还贷，离婚怎么分？”

回答结构：相关法条摘要 + 通俗解释 + 相似案例参考 + 下一步建议（协商/调解/诉讼）+ 证据材料清单。

支持多轮对话记忆，上下文关联。

4.2 启发式主动引导（防卡死）
当用户表达“不知道”“不明白”“说不清”时，或系统检测到关键信息缺失，必须根据已有上下文生成 2-3 个专业选项，供用户点击选择。选项应涵盖法律分析中的关键维度（主体关系、事实经过、争议焦点、诉求目的），并附带“都不是，我再说说”的开放式兜底。

详细策略见第 8 节。

4.3 案例检索与“以案说法”（RAG）
从案例库中检索与用户描述最相似的脱敏案例。

回答时引用案例：包含案件摘要、判决结果、案号、法院，并强调“个案不同，仅供参考”。

案例来源：最高人民法院公报案例、指导性案例、中国裁判文书网公开判例（脱敏处理）。

4.4 法律法规检索
支持关键词、法规名称等检索《民法典》婚姻家庭编、继承编及相关司法解释。

展示条款原文并附通俗解读。

4.5 计算辅助
离婚案件诉讼费估算：根据标的额计算。

抚养费参考计算：基于对方月收入的 20%-30% 及当地消费水平估算。

房产分割估算：结合出资比例、还贷情况给出计算思路（注明仅供参考）。

4.6 反家暴工具包
一键展示：报警指引、验伤流程、证据固定方法、当地庇护所/法律援助中心查询（接入地图 API 或手动录入）。

4.7 用户反馈
每个回答下方提供“有帮助 / 没帮助 / 违规”按钮，用于持续优化。

Embedding 模型：BAAI/bge-large-zh-v1.5 或同类中文法律优化模型。

6.2 案例库（关键）
每个案例存储结构：

  "case_id": "唯一标识",

数量要求：MVP 阶段至少准备 200 个高质量婚姻家事案例，覆盖离婚、财产、抚养、家暴、继承等子类。

6.3 结构化问答对
收集 12348 常见问答、法宣案例，人工封装成“问题-答案”对，增强小范围问答准确率。

7. 提示词工程 (System Prompt 设计)
这是 LLM 运行时的基座提示，必须内置：

text
你是一位专业的婚姻家事法律知识助手，仅能基于给定的参考资料提供普法信息，不能提供法律意见。

规则：
1. 回答先引用相关法条原文，然后用通俗语言解释。
2. 若检索到相似案例，请以“类似案例参考”形式引用，包括案件摘要、案号和判决结果。
3. 不能预测案件结果，必须强调“每个案件情况不同，案例仅供参考”。
4. 家暴、人身安全威胁：立刻在回答首屏给出报警指引和求助热线。
5. 涉及操作流程（如起诉、协议离婚），必须建议咨询律师或拨打 12348。
6. 所有回答结尾必须包含：“以上内容仅供参考，不构成法律意见。”
7. 不知道答案时明确说明，绝不编造法条或案号。
8. 语气专业、温和、有同理心，但保持法律中立。
8. 启发式选项机制详情（防卡死核心）
8.1 触发条件
用户输入包含：不明白 听不懂 什么意思 没懂 不知道怎么说 说不清 很难描述

用户对上一轮回答点击“没帮助”或回复“不对”“不是这个”

系统主动追问后，用户回复仍缺乏关键实体（由后端槽位状态判断）

8.2 选项生成策略
调用 LLM 生成 2-3 个选项，Prompt 如下：

text
当前用户遇到理解或表达困难。请根据对话历史，生成2-3个专业且易懂的选项，帮助用户推进咨询。
要求：
- 严格基于已描述的事实，不编造。
- 若用户不解术语，提供解释选项+行动选项。
- 若案情模糊，将他的模糊表达映射到最可能的2-3个婚姻家事法律场景。
- 每个选项不超过20字，最后固定一个“都不是，我再说说”选项。
- 输出JSON：{"options": ["选项A", "选项B", "选项C"]}

对话历史：{history}
用户最近输入：{user_input}
已知实体：{entities}
8.3 前端渲染
后端返回 JSON，前端将选项渲染为可点击按钮，点击即作为新消息发送。

8.4 特殊引导实例（内置模板）
情绪转译：用户“我对他彻底失望了” → 追选项：A. 有过肢体冲突报警 B. 发现转移财产 C. 长期分居 D. 先了解离婚流程

证据清单：用户想证明感情破裂 → 选项：A. 家暴报警回执 B. 出轨聊天记录 C. 分居满两年证明 D. 帮我梳理证据

9. 案例 RAG 集成细节
在接收到用户问题后，同时进行：

法条向量检索（检索域：法条 chunk）
案例向量检索（检索域：案例 facts_summary + judgment_result）
案例检索需结合关键词过滤（如 dispute_type）和相似度阈值（推荐 0.75）。

指导性案例优先展示。

检索到的法条和案例一并嵌入对话 Prompt，要求 LLM 按第 7 节规则生成回答。

10. 对话管理
使用数据库或内存维护 session_id 及以下信息：

最近 5 轮对话文本

当前对话状态（正常 / 困惑状态 / 等待槽位填充）

已识别槽位：marital_status, has_children, property_type, violence_involved, fault_party 等

当用户切换话题到非婚姻家事时，给出引导并询问是否继续原话题。

11. 安全与合规
数据隐私：用户对话涉及案情时，必须在服务端做匿名化处理或脱敏；不存储个人身份信息。

版权：法条原文不受著作权保护；案例摘要需自行撰写，不得全文搬运商业数据库内容。

内容安全：另设关键词黑名单及输出审核，防止输出煽动、诽谤、虚假信息。

免责：每次回答结尾强制显示“以上内容仅供参考，不构成法律意见”。

12. 里程碑验收标准
启发式选项：连续 5 次用“我不明白”或模糊描述提问，系统均能给出合理选项且不重复卡死。

案例引用：对高频婚姻问题，回答准确引用至少一个相似案例，并附案号。

拒答准确率：对 20 个非婚姻法律问题或闲聊，拒答率 > 95%，且不会误拒“你是谁”等白名单查询。

安全响应：涉及家暴的提问，回答首屏必有报警指引。

合规声明：每次回答结尾包含免责声明。
