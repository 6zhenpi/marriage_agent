# Collected legal data - 采集法律数据存储

## 目录用途

本目录存储婚姻家庭法律咨询 Agent 项目通过 Snail 爬虫采集的所有原始法律数据。所有文件必须经过 `Snail/utils/validator.py` 验证后方可存入。

## 目录结构

```
Collected legal data/
├── README.md                # 本文件
├── metadata.json            # 元数据索引（自动生成，勿手动编辑）
├── _rejected/               # 被拒绝的文件（格式不符等）
├── 法律条文/                  # A类：法律、行政法规原文
├── 司法解释/                  # B类：最高法司法解释、批复、指导意见
├── 裁判案例/                  # C类：指导性案例、参考案例、典型判例
├── 实务指引/                  # D类：诉讼流程、证据清单、文书模板
└── 常见问答/                  # E类：12348热线高频问答
```

## 各子目录内容说明

| 子目录 | 数据内容 | 目标数量 | 优先级 |
|--------|---------|---------|--------|
| 法律条文 | 民法典婚姻家庭编、继承编、反家暴法、妇女权益保障法等 | 50+ 条 | 核心 |
| 司法解释 | 民法典婚姻家庭编解释(一)(二)、继承编解释(一)、彩礼司法解释等 | 3-5 部 | 核心 |
| 裁判案例 | 离婚财产分割、抚养权、家暴、彩礼、继承等案例 | 150-200 条 | 核心 |
| 实务指引 | 起诉流程、证据清单、保护令申请指南等 | 20-30 条 | 重要 |
| 常见问答 | 婚姻家庭领域高频法律问答 | 30-50 条 | 补充 |

## 文件命名规则

| 规则 | 格式 | 示例 |
|------|------|------|
| 法律条文 | `{法律简称}_{条号}.{ext}` | `民法典_第1041条.txt` |
| 司法解释 | `{司法解释简称}_全文.{ext}` | `婚姻家庭编解释一_全文.txt` |
| 裁判案例 | `{案号}_{争议类型简称}.{ext}` | `2023京0105民初1234_财产分割.txt` |
| 实务指引 | `{指引类型}_{简要描述}.{ext}` | `起诉流程_离婚诉讼.txt` |
| 常见问答 | `FAQ_{序号}_{关键词}.{ext}` | `FAQ_001_彩礼返还.txt` |

## 格式要求

### 允许的文件格式（仅限以下三种）

| 格式 | 扩展名 | 用途 | 说明 |
|------|--------|------|------|
| 纯文本 | `.txt` | 首选格式 | UTF-8 编码，结构化文本 |
| PDF | `.pdf` | 原始文书 | 保留原始排版 |
| Word | `.docx` / `.doc` | 原始文书 | 保留原始排版 |

### 格式验证机制

1. **扩展名检查**：仅接受 `.pdf`、`.txt`、`.doc`、`.docx` 四种扩展名
2. **文件头校验**：通过 Magic Bytes 验证文件真实类型（防止扩展名伪造）
3. **空文件检测**：拒绝 0 字节文件
4. **被拒绝文件**：自动移入 `_rejected/` 子目录，并在 `metadata.json` 中记录拒绝原因

## 元数据文件 (metadata.json)

每条记录包含以下字段：

```json
{
  "filename": "民法典_第1041条.txt",
  "category": "法律条文",
  "source": "国家法律法规数据库",
  "original_path": "C:\\temp\\download\\xxx.txt",
  "saved_path": "d:\\project\\marriage_agent\\Collected legal data\\法律条文\\民法典_第1041条.txt",
  "file_size_bytes": 1024,
  "file_extension": ".txt",
  "collected_at": "2026-04-27T10:30:00",
  "status": "accepted"
}
```

被拒绝的文件记录：

```json
{
  "filename": "image.png",
  "category": "法律条文",
  "source": "xxx",
  "original_path": "C:\\temp\\download\\image.png",
  "file_extension": ".png",
  "rejection_reason": "Unsupported file extension",
  "rejected_at": "2026-04-27T10:30:00",
  "status": "rejected"
}
```

## 特殊处理说明

1. **禁止手动放置文件**：所有文件必须通过 `Snail/utils/validator.py` 的 `save_to_collected_data()` 函数写入，以确保格式验证和元数据记录
2. **数据脱敏**：裁判案例中的当事人姓名、身份证号、详细住址等个人信息必须在采集阶段完成脱敏
3. **版权标注**：每个文件开头应注明数据来源和采集日期
4. **编码统一**：所有 `.txt` 文件使用 UTF-8 编码
5. **metadata.json 禁止手动编辑**：该文件由验证模块自动维护
